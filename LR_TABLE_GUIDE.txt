برای گزارش Word:
────────────────────────────────────────────────────────────────────────────────────
1. باز کردن LR_TABLE_COMPACT.txt
2. کپی کردن جدول
3. Paste کردن در Word با فونت Courier New یا Consolas
4. تنظیم اندازه فونت (8-10 pt)

نکات Word:
  • فونت: Courier New یا Consolas
  • اندازه: 8-10 pt
  • راستچین کردن: Center
  • حاشیه: ۱ سانتی‌متر


برای گزارش PDF/LaTeX:
────────────────────────────────────────────────────────────────────────────────────
1. استفاده از LR_PARSING_TABLE_ASCII.txt
2. قرار دادن در محیط verbatim یا lstlisting

مثال LaTeX:
  \begin{verbatim}
  [محتوای جدول]
  \end{verbatim}

4. یک اسلاید برای جدول GOTO

🎨 توصیه‌های طراحی:
═══════════════════════════════════════════════════════════════════════════════════════

برای گزارش:
  ✓ استفاده از فونت Monospace (Courier New, Consolas)
  ✓ حجم فونت 8-10 برای جدول کامل، 10-12 برای فشرده
  ✓ حاشیه کم برای جدول عریض
  ✓ قرار دادن جدول در صفحه جداگانه (Landscape)

برای ارائه:
  ✓ تقسیم به 2 اسلاید (ACTION + GOTO)
  ✓ هایلایت state های مهم (0, 2, 8)
  ✓ استفاده از animation برای نمایش مثال
  ✓ رنگ‌بندی سلول‌ها برای بهتر دیدن

برای چاپ:
  ✓ صفحه Landscape (افقی)
  ✓ حاشیه 1cm از هر طرف
  ✓ فونت سیاه (بدون رنگ)
  ✓ خطوط واضح


═══════════════════════════════════════════════════════════════════════════════════════
💡 نکات مهم:
═══════════════════════════════════════════════════════════════════════════════════════

1. حفظ نسبت عرض به ارتفاع:
   جدول‌های ASCII برای نمایش درست نیاز به فونت Monospace دارند.

2. جدول ACTION vs GOTO:
   • ACTION: عملیات shift/reduce روی terminal ها
   • GOTO: انتقال روی non-terminal ها

3. خلاصه‌سازی:
   در گزارش می‌توانید فقط state های کلیدی را توضیح دهید:
   - State 0: شروع
   - State 2: بعد از mnemonic (تصمیم‌گیری)
   - State 8: بعد از REGISTER (با/بدون offset)
   - State 1: Accept

4. مثال‌ها:
   همیشه یک trace کامل در گزارش بگنجانید (مثل CLFLUSHOPT [EBX+16]).

🎓 پرسش‌های متداول:
═══════════════════════════════════════════════════════════════════════════════════════

Q: آیا باید همه 17 state را توضیح دهم؟
A: خیر. 3-4 state کلیدی کافی است (0, 2, 8, 14).

Q: جدول در Word خراب می‌شود؟
A: از فونت Courier New یا Consolas استفاده کنید و اندازه را کم کنید.

Q: جدول خیلی عریض است؟
A: از نسخه Compact استفاده کنید یا صفحه را Landscape کنید.

Q: آیا باید جدول را رنگی کنم؟
A: برای گزارش چاپی، سیاه-سفید بهتر است. برای ارائه، رنگ کمک می‌کند.

Q: کدام فرمت برای گزارش بهتر است؟
A: LR_TABLE_COMPACT.txt برای Word، LR_PARSING_TABLE_ASCII.txt برای PDF.
